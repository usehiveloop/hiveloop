Take a screenshot of a webpage. Returns a base64-encoded PNG image.

## When to Use
- Capture the visual state of a webpage for review or comparison
- Verify that a deployed page renders correctly
- Document the current appearance of a UI
- Capture visual evidence of a bug or layout issue

## Parameters
- `url` (required): The URL to screenshot
- `request`: Rendering mode — "http" (basic), "chrome" (full browser rendering, recommended), "smart" (auto-detect). Use "chrome" for accurate visual screenshots
- `wait_for_selector`: Wait for a specific element before capturing. Useful for pages that load content dynamically
  - `selector`: CSS selector to wait for (e.g. ".main-content", "#app", "[data-loaded]")
  - `timeout`: Maximum wait time in milliseconds

## Tips
- Always use `request: "chrome"` for visual accuracy — HTTP-only rendering won't execute JavaScript or apply CSS fully
- Use `wait_for_selector` for SPAs that load content after the initial page load
- The returned image is base64-encoded PNG — suitable for embedding in reports or passing to vision models
