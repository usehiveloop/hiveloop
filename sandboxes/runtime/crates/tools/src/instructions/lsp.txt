Code intelligence via the Language Server Protocol. Use after editing to check for compile errors, and before editing to understand structure.

## Operations
- `goToDefinition` (line, character): jump to symbol definition
- `findReferences` (line, character): list all usages
- `hover` (line, character): type info and docs at position
- `documentSymbol`: list symbols in `file_path` (no position)
- `workspaceSymbol` (query): search symbols project-wide; up to 10 results
- `goToImplementation` (line, character): find impls of interface/trait
- `prepareCallHierarchy` / `incomingCalls` / `outgoingCalls` (line, character): callers/callees
- `diagnostics`: errors/warnings for `file_path` (no position)

## Parameters
- `operation` (required), `file_path` (required, abs or relative)
- `line`, `character`: 1-based; required for position-based ops
- `query`: only for `workspaceSymbol`

## Notes
Files are opened/refreshed automatically. If no server matches the file type, returns an error gracefully. Use `diagnostics` after edits to verify; use `goToDefinition` / `findReferences` instead of grep when navigating typed code.
