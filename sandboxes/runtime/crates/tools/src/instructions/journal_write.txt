Append an entry to the conversation journal — a persistent log that survives history stripping and context resets.

## Why
Old tool results get stripped after a protection window. Old assistant text can also disappear across chain resets. If you'll need a specific value later (a path, a count, a decision rationale), journal it NOW. Restating it in your own text is not enough.

## When to use
Record high-signal items only:
- Decisions ("we use PostgreSQL because…")
- Discoveries ("auth module has a race in session refresh")
- User preferences and blockers
- Specific values from tool output you'll reference later

## When NOT to use
Skip routine progress, large code snippets (use file paths instead), and anything only relevant to the current turn.

## Format
Keep entries 1-3 sentences. Use `category` to tag: `decision` / `discovery` / `blocker` / `progress` / `preference`. Aim for ~2-5 entries per session, not every turn — focus on the "why" not the "what".
