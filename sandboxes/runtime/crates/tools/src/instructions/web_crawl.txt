Crawl a website starting from a URL, following links to discover and return page content.

## When to Use
- Crawl documentation sites to gather multiple pages of content
- Scrape product listings, blog posts, or knowledge bases
- Discover and retrieve content across an entire site section
- Gather training data or research material from web sources

## Parameters
- `url` (required): The starting URL to crawl from
- `limit`: Maximum number of pages to crawl. Set to 1 to fetch a single page. Omit for no limit (use with caution — prefer setting a limit)
- `depth`: Maximum link depth from the starting URL. 0 means no limit
- `return_format`: Output format — "markdown" (default, best for reading), "raw" (original HTML), "text" (plain text), "html2text" (simplified HTML)
- `request`: Rendering mode — "http" (fast, static pages), "chrome" (JavaScript-heavy SPAs), "smart" (auto-detect, recommended for unknown sites)
- `readability`: When true, extracts clean article content and removes navigation, ads, and boilerplate

## Tips
- Always set a `limit` to avoid crawling thousands of pages accidentally
- Use `request: "smart"` when you're unsure whether a site uses JavaScript rendering
- Use `return_format: "markdown"` for content you need to read or summarize
- Set `depth: 1` to only follow links one level deep from the starting page
- For a single page fetch, set `limit: 1`
