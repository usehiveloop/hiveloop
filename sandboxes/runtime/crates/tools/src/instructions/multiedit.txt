Apply multiple find-and-replace edits to a single file in one atomic operation. Prefer over `edit` when changing multiple parts of the same file.

## Parameters
- `file_path`: absolute path
- `edits`: array of `{ oldString, newString, replaceAll? }` (oldString must match file contents exactly, including whitespace)

## Semantics
Edits run in order; each operates on the result of the previous one. All-or-nothing — if any edit fails, none are applied. The file must have been Read first.

Use `replaceAll: true` to rename a variable (or any identifier) across the file. Plan ordering so earlier edits don't move text that later edits search for.
