PRECONDITIONS, ASSUMPTIONS, AND CONTRACTS
-----------------------------------------

These are the load-bearing assumptions internal/hermes/compile.go and the
sidecar SyncRequest contract bake in today. If any of these change, the
corresponding code must change with them.

ORCHESTRATOR-INJECTED ENV (sidecar will not boot without these)
  - CONTROL_PLANE_URL          : base URL the sidecar dials for upstream sync
                                 callbacks (skills, sessions, memory, cron).
  - CONTROL_PLANE_API_KEY      : bearer token the sidecar uses on every CP
                                 callback.
  - AGENT_ID                   : UUID; sidecar tags every upstream payload
                                 with this so CP can route to the right row.
  - HIVELOOP_GIT_USERNAME      : becomes `git config --system user.name`
                                 inside the sandbox. Without it, commits are
                                 unattributed.
  - HIVELOOP_GIT_EMAIL         : becomes `git config --system user.email`.
  - HIVELOOP_GIT_CREDENTIALS_URL : CP endpoint that vends short-lived GitHub
                                 installation tokens scoped to the agent's
                                 allowed repos. Used by the baked-in
                                 git-credential-hiveloop and gh-wrapper
                                 helpers. Without it, push/PR ops fail.

CP PRECONDITIONS (must be true before Compile() is called)
  - Hindsight bank `bank-{org_id}` exists. Compile emits the bank_id but
    does NOT create the bank. CP creates it during org provisioning. If
    missing, Hermes will boot but memory writes will fail.
  - agent.OrgID is non-nil (Compile errors otherwise).
  - A usable Credential resolves via credentials.Picker — either a tenant-
    owned cred for the agent's selected provider, or a system fallback.
  - SigningKey is the same key the proxy uses to verify ptok_* — they MUST
    match or every LLM call from inside the sandbox 401s.
  - cfg.ProxyHost points at proxy.usehiveloop.com (or equivalent), and cfg
    .MCPBaseURL points at the hiveloop integration MCP endpoint.

SOUL.md / SYSTEM PROMPT
  - The file written to /opt/data/SOUL.md is the EXACT, VERBATIM contents
    of agent.SystemPrompt — no header, no name/category/role wrapper, no
    template substitution. Whatever the user authored in the CP UI is what
    Hermes reads on boot.
  - Hermes loads SOUL.md as the system prompt verbatim. There is no
    runtime mutation between Compile and Hermes' first turn.
  - System prompts are NOT versioned at the SkillVersion level — they live
    on Agent itself, so editing a prompt is immediately visible at the
    next sync.

PROVIDER DISPATCH (compile.go::hermesProviderForCredential)
  - Only providers in `hermesAnthropicProviders` get `provider: anthropic`
    + bare-host base_url (the Anthropic SDK will append /v1/messages
    itself). Today that set is { "anthropic": true }.
  - Everything else gets `provider: custom` + base_url ending in /v1 (the
    OpenAI SDK in Hermes will append /chat/completions, /embeddings, etc.).
  - This assumes every non-Anthropic upstream behind the proxy exposes an
    OpenAI-compatible chat-completions surface. Bedrock-Anthropic /
    Vertex-Anthropic / native Anthropic-shape upstreams are NOT yet listed
    and will currently be sent down the OpenAI path — broken until added.

PTOK / TOKEN PERSISTENCE
  - Every Compile() mints a fresh proxy token with 24h TTL and writes a
    model.Token row tagged Meta.harness="hermes". The proxy authenticates
    ALL traffic from the sandbox (LLM calls, MCP calls) against this row.
  - Re-running Compile() does NOT revoke the prior token; both will be
    valid until natural expiry. Token cleanup is the proxy's job.
  - Scopes are derived from agent.Integrations[].actions; if integrations
    is empty the token has no per-tool scopes (proxy is permissive on
    LLM calls regardless).

ENV FILE (compile_env.go)
  - .env is written to /opt/data/.env at mode 0600.
  - OPENAI_API_KEY and CUSTOM_API_KEY are both set to the ptok_*; this
    covers both SDK code paths in Hermes that look for one or the other.
  - agent.EncryptedEnvVars is decrypted via deps.EncKey (NOT the KMS
    envelope path — these are SymmetricKey-encrypted JSON), then keys
    starting with `BRIDGE_` are filtered out (those belong to the bridge
    harness only).
  - AgentProfile secrets are decrypted via the KMS envelope: KMS.Unwrap
    DEK → DecryptCredential → mergeProfileSecrets. Today only Slack is
    handled (emits SLACK_BOT_TOKEN, SLACK_APP_TOKEN, SLACK_ALLOWED_USERS=*).
  - Output is sorted alphabetically for deterministic diffs.

SKILLS TREE
  - One directory per skill at skills/<slug>/, where <slug> comes from the
    SkillVersion. Each dir contains SKILL.md (with synthesized frontmatter
    `name: <slug>` + `description: ...`) plus every entry in
    bundle.Files at its declared relative path.
  - Skills resolve in this order: pinned SkillVersion on AgentSkill first;
    otherwise latest published SkillVersion.
  - Skills written by the agent at runtime are out-of-band: the sidecar
    picks them up via fsnotify and pushes upstream. Compile only seeds
    initial state.

REPOS
  - buildRepoSpecs reads agent.Resources, looks for entries of type
    "repository", and emits {url, path, branch, ref} per repo.
  - URL is hard-coded `https://github.com/<id>.git`. Assumes the resource
    `id` is `<owner>/<repo>` and the host is github.com. GitLab/Bitbucket/
    self-hosted/ssh URLs are NOT supported.
  - `ref` is currently never populated (P2 item above).
  - Sidecar clones with --depth 1 into /opt/data/workspace/<path>. On
    subsequent syncs it fetches+resets to the latest target. Local edits
    in the sandbox are NOT preserved across re-clones — agent commits
    must be pushed before any re-sync.

CONFIG.YAML SHAPE (the contract Hermes parses)
  - agent.max_turns is hard-coded to 180 (raised from 50; "we're using a
    big model"). agent.AgentConfig override fields are NOT yet read.
  - terminal.backend is hard-coded "local".
  - memory provider is hard-coded "hindsight"; bank scope is org-wide
    (bank-{org_id}). Per-employee isolation is a v2 item.
  - platform_toolsets.api_server is the static list:
    {terminal, file, skills, todo, memory, session_search, vision, cronjob}.
    No web tools; web fetch/search will arrive as MCP servers later.
    agent.Tools and agent.Permissions are IGNORED (v2 item).
  - mcp_servers contains exactly one entry today: the hiveloop integration
    MCP, addressed at {cfg.MCPBaseURL}/{jti} with Authorization: Bearer
    ptok_*. Per-Hermes-agent MCP tools (spider, hermes-mgmt) are a P2.

SYNC SEMANTICS
  - Compile always returns FullSync=true. Sidecar treats the file list as
    the full desired state for paths it manages — anything CP-managed not
    in the list is removed. Files outside the CP-managed set (skills the
    agent created, sessions, runtime caches) are preserved.
  - Version = agent.UpdatedAt.UnixNano(). Sidecar rejects any sync whose
    version <= last applied. Touching agent.UpdatedAt is required to
    force a re-sync.
  - The sidecar will NOT start Hermes until the first SyncConfig POST
    succeeds. State `awaiting_initial_config` blocks /v1/hermes/restart
    with 503. CP must push at least once after sandbox boot.

WHAT IS DELIBERATELY NOT PUSHED
  - No setup_commands (v2).
  - No per-agent Hermes plugins.
  - No per-agent hooks.
  - No proxy URL for Hindsight (sidecar is expected to talk to Hindsight
    directly today; CP-token-authed proxy is a P2).
  - No per-employee memory bank scope.
  - No web fetch/search tool config (intentional — disabled in Hermes,
    will be added via MCP later).
